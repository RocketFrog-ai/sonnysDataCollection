# Zeta modelling — test split hit/miss report

## Important: why test sites ≈ total sites

This report scores **only the test period** (`date >= 2025-01-01`). **No 2024 (train) months** are included in actual, predicted, or annual sums.

The split is **by calendar time**, not by holding out random sites:

| | What it means |
|---|---|
| **Train** | All sites’ months in **2024** (Model 1: 10,462 rows) |
| **Test** | The **same sites’** months in **2025** (Model 1: 8,403 rows) |
| **Model 1** | Panel has **882 sites total** → **882 sites in test** = **100% of sites**, because every site has both 2024 train and 2025 test months |
| **Site overlap** | 882 sites appear in **both** train and test (expected for a temporal split) |
| **NOT included** | Train-period volumes (2024) are **never** summed into “actual” or “predicted” here |

**Verified:** `model1_benchmark_volume_sample.csv` has **0** rows before `2025-01-01` and **8,403** rows = exactly the panel’s test slice.

If you expected fewer test sites (e.g. 20% site holdout), that is a **different** evaluation design. Zeta models use **2024 → train, 2025 → test** on the same site universe.

---

## Scoring rules

- **Hit:** `|actual − predicted| ≤ 25,000`
- **Monthly table:** each row is one **test month** (2025 only)
- **Annual site table:** sum of **test months only** per site (`n_test_months` shows how many 2025 months that site has; 376 Model 1 sites have &lt; 12 months)
