================================================================================
QUANTILE ANALYSIS — METHODOLOGY & FEATURE REFERENCE
================================================================================
(scoringmetric/approach2/quantile_predictor.py)

Data source: Car wash counts from v2/temp_extrapolated.csv; features from
Proforma Excel. Merge on street + 5-digit ZIP. See QUANTILE_FLOW.md for full flow.

--------------------------------------------------------------------------------
1. CORE IDEA
--------------------------------------------------------------------------------

  For every new input location we answer two questions:

  QUESTION 1 — CAR WASH COUNT PREDICTION:
    "Which of the four car wash volume tiers (Q1–Q4) does this location belong to?"
    We predict this using 19 environmental/location features and a trained classifier.

  QUESTION 2 — FEATURE CORRELATION WITH CAR WASH COUNT:
    "For each of the 19 features, does this location's value match what Q1, Q2, Q3,
    or Q4 car wash sites typically look like?"

  The car wash COUNT is the ground truth. Grouping sites by count into Q1–Q4 lets
  us ask: "among the top-25%-volume car wash sites, what is their typical snowfall,
  sunshine, competitor count, etc.?"  A new location's feature value is then mapped
  to whichever car wash volume group it most resembles.

--------------------------------------------------------------------------------
2. CAR WASH COUNT QUANTILES (THE TARGET) — EQUAL COUNT OF SITES
--------------------------------------------------------------------------------

  Strategy: Quartiles based on an EQUAL COUNT of sites (e.g. bottom 25% of N sites,
  next 25%, etc.) so tiers are business-explainable: "bottom 25%", "top 25%".

  Step 1: Take actual annual car wash counts (current_count) from temp_extrapolated
          after merge with Proforma features. N = number of sites with both.
  Step 2: Apply equal-frequency percentile boundaries (0, 25, 50, 75, 100):
            Q1 = bottom 25% of sites by count  (lowest volume, ~N/4 sites)
            Q2 = next 25%
            Q3 = next 25%
            Q4 = top 25% of sites by count     (highest volume, ~N/4 sites)
  Value ranges (e.g. Q1 narrow, Q4 wide) will differ; each tier has ~same # of sites.

  These four groups become both the target variable for prediction AND the
  reference profiles used in feature correlation analysis.

  A new location has no measured count. We PREDICT which group it belongs to
  from the 19 features (see Section 5).

--------------------------------------------------------------------------------
3. FEATURE CORRELATION WITH CAR WASH COUNT  ("WashQ")
--------------------------------------------------------------------------------

  This is the key addition over a simple feature percentile rank.

  For each feature (e.g. Annual Sunshine Hours):
    a) Take Q1 car wash sites → compute their median sunshine:  3305 h
       Take Q2 car wash sites → compute their median sunshine:  3328 h
       Take Q3 car wash sites → compute their median sunshine:  3335 h
       Take Q4 car wash sites → compute their median sunshine:  3349 h

    b) These four medians are the "profile" of each performance tier.

    c) For a new location with sunshine = 3850 h:
       - Direction-adjust: sunshine is "higher is better", so keep values as-is.
       - Find which group median is closest: |3850 − 3349| = 501 (Q4 is closest).
       - WashQ = Q4. This means: "3850h of sunshine is most typical of Q4
         (high-performing) car wash sites."
       - If the value EXCEEDS even the Q4 group median, flag "exceeds Q4 ★".

    d) For "lower is better" features, direction-adjust by inverting (multiply by −1)
       before computing distance to group medians, so "better than Q4" still means
       WashQ = Q4.

  WashQ answers: "Does this feature value look like a Q1 car wash location,
  a Q2 one, or a Q4 one?"

  Note: features with weak correlation to car wash count will have nearly
  identical group medians across Q1–Q4. In those cases WashQ is uninformative
  (the value could land anywhere). The "Pctile" column still shows how this
  value compares to all training sites.

--------------------------------------------------------------------------------
5. HOW WASH COUNT QUARTILE IS PREDICTED FOR A NEW LOCATION
--------------------------------------------------------------------------------

  Model:  Random Forest classifier (300 trees, class_weight='balanced')
  Input:  19 feature values (missing values → filled with training-set median)
  Output: Predicted wash_q in {1, 2, 3, 4} + probability for each class

  "Model confidence" = probability assigned to the predicted class.
  E.g. "Q4 with 37% confidence" means 37% of the 300 trees voted Q4.

  CV accuracy (5-fold cross-validation):
    Exact (right quartile):   ~35–38%   vs 25% random baseline
    Within-1-quartile:        ~71–73%   (target: >70%)

  Model may be rebuilt using only control sites (exclude test/training sites) and
  with additional features to improve toward >70% within-1-quantile accuracy.

  Why not higher accuracy?
    The 19 features are environmental/location-only. Car wash volume is also
    driven by pricing, operating hours, marketing, and traffic patterns — none
    of which are in these features. The model does the best it can with what
    is available.

--------------------------------------------------------------------------------
6. FEATURE QUANTILE (ADJUSTED PERCENTILE — FOR REFERENCE)
--------------------------------------------------------------------------------

  In addition to WashQ, we still report "Pctile" — the adjusted percentile of
  this location's value across ALL 1064 training sites for that feature:

    raw percentile  = % of all sites this value beats on this feature
    adj percentile  = direction-corrected: for "lower is better" features,
                      adj_pct = 100 − raw_pct so that 90% still means excellent

  This tells you: "You beat 90% of all car wash sites on sunshine hours."
  WashQ tells you: "That sunshine level matches what Q4 car wash sites have."

  Both pieces of information together give the full picture.

--------------------------------------------------------------------------------
7. WHAT EACH COLUMN IN THE REPORT MEANS
--------------------------------------------------------------------------------

  Column      | Meaning
  ------------|------------------------------------------------------------------
  Value       | This location's raw feature value
  WashQ       | Which car wash volume tier (Q1–Q4) this feature value matches,
              | based on proximity to that tier's median feature value.
              | ★ = exceeds even Q4 typical.  ▲ = matches Q4.  ▼ = matches Q1.
  Pctile      | Adjusted percentile vs ALL 1064 training sites (direction-adj).
              | 90% = you beat 90% of all sites on this feature.
  Q4 median   | The median value of this feature among Q4 (top-25% volume)
              | car wash sites. This is the "high performer benchmark."
  Importance  | Feature importance in the Random Forest model: how much this
              | feature influences the wash-count-quartile prediction.

  VISUALIZATION (each feature histogram panel):
    Dashed coloured lines = group median for Q1 / Q2 / Q3 / Q4 car wash sites.
    Solid coloured line   = this location's value.
    Colour of solid line  = matched WashQ (red=Q1, orange=Q2, green=Q3, blue=Q4).
    If solid line is to the right of the blue dashed line → exceeds Q4 (★).

--------------------------------------------------------------------------------
8. STRENGTHS & WEAKNESSES (INVESTMENT-REPORT STYLE)
--------------------------------------------------------------------------------

  The report includes an investment-style summary:
    Strengths: Features where this location matches or exceeds Q4 (high-performing)
               sites' typical values (WashQ = Q4 or exceeds Q4).
    Weaknesses: Features where this location matches Q1 or Q2 (low or below-median)
                performance tier.
